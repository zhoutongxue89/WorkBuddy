阶段 1 · 环境搭建 & 入门起步（第1~2周）
🛠️ 开发环境安装
  • 去 python.org 下载 Python 3.12+
  • 安装时勾选 "Add to PATH"
  • 终端输入 python --version 验证
  • 安装 VS Code + Python 插件
  • 安装 Pylance、Code Runner 插件
  • 了解 Jupyter Notebook（数据方向必备）
📝 第一个程序
  • 认识 .py 文件
  • print() 输出语句
  • input() 获取用户输入
  • 注释：# 单行 / """ 多行
  • 缩进规则（Python 的灵魂）
  • 了解解释型语言 vs 编译型语言
🔢 数据类型入门
  • 整数 int：1, 100, -5
  • 浮点数 float：3.14, -0.5
  • 字符串 str："Hello", 'Python'
  • 布尔值 bool：True / False
  • type() 查看类型
  • 变量命名规范（下划线命名法）
🧮 运算符
  • 算术：+ - * / // % **
  • 比较：== != > < >= <=
  • 逻辑：and or not
  • 赋值：= += -= *= /=
  • 字符串拼接与重复
  • f-string 格式化输出
💡 入门阶段学习建议
  • 每天保证 1~2 小时练习
  • 手敲每一行代码，不要复制粘贴
  • 出错是正常的，看报错信息解决问题
  • 先跑通，再理解原理
  • 用 Python 解决生活小问题（如算账）
  • 不要纠结细节，保持向前推进
示例：人生第一个Python程序
# 变量定义 name = input("请输入你的名字: ") age = int(input("请输入你的年龄: ")) # f-string 格式化输出 print(f"你好，{name}！你今年 {age} 岁，{2026 - age} 年出生。") # 简单计算 years_to_30 = 30 - age if years_to_30 > 0: print(f"距离30岁还有 {years_to_30} 年，加油！") else: print("你已经超过30岁了，经验丰富！")

阶段 2 · Python 基础语法（第3~6周）
🔀 控制流
  • if / elif / else 条件判断
  • for 循环 + range()
  • while 循环
  • break / continue / pass
  • 嵌套循环
  • match-case（Python 3.10+）
📦 数据容器
  • 列表 list：有序可变 [1,2,3]
  • 元组 tuple：有序不可变 (1,2,3)
  • 字典 dict：键值对 {"key": val}
  • 集合 set：无序不重复 {1,2,3}
  • 常用方法：append/pop/update
  • 嵌套容器与遍历
🔧 函数
  • def 定义函数

  • 可变参数 *args **kwargs
  • return 返回值（多返回值）
  • lambda 匿名函数
  • 作用域：局部/全局 global
✂️ 字符串进阶
  • 切片 s[start:end:step]
  • split / join / strip
  • replace / find / count
  • upper / lower / title
  • startswith / endswith
  • 正则表达式 re 模块基础
📁 文件操作
  • open() 打开文件
  • read / readline / readlines
  • write / writelines
  • with 上下文管理器（推荐）
  • 文件路径：os.path 模块

🧩 常用内置函数
  • len / range / enumerate / zip
  • map / filter / sorted / reversed
  • max / min / sum / abs / round
  • list / tuple / dict / set / str
  • isinstance / type
  • help() 查看文档
示例：综合运用基础语法
# 统计一段文字中每个单词出现次数 def word_count(text): words = text.lower().split() counter = {} for word in words: counter[word] = counter.get(word, 0) + 1 return counter text = "python is great python is easy to learn" result = word_count(text) # 按出现次数排序输出 for word, count in sorted(result.items(), key=lambda x: x[1], reverse=True): print(f" {word:10} → {count} 次")

示例：面向对象设计一个银行账户
class BankAccount: bank_name = "Python银行" # 类属性 def __init__(self, owner, balance=0): self.owner = owner self._balance = balance # 下划线表示私有 @property def balance(self): return self._balance def deposit(self, amount): if amount <= 0: raise ValueError("存款金额必须大于0") self._balance += amount return self def __str__(self): return f"{self.bank_name} | {self.owner} | 余额: ¥{self._balance:,.2f}" acc = BankAccount("张三", 1000).deposit(500).deposit(200) print(acc) # Python银行 | 张三 | 余额: ¥1,700.00

新手常见误区 & 避坑指南
❌ 光看不练，只看视频刷完就觉得会了
✅ 每学一个知识点立刻动手敲代码，哪怕只是改改例子跑一跑。编程是手艺，必须练。
❌ 遇到 Bug 就崩溃，一看报错就放弃
✅ 报错是宝贵信息！认真读错误信息的最后一行，90% 的问题都在那里。调试是必学技能。
❌ 同时学多个方向/框架，什么都想会
✅ 先打牢基础，专注一个方向。Django + 数据库 OR Pandas + 可视化，一次只选一个。
❌ 跳过基础直接学框架（如直接学 Django）
✅ 没有扎实的基础，学框架只会云里雾里。先把语法、OOP、异常处理学透再上框架。
❌ 不写项目，只做练习题就满足了
✅ 做完基础后马上做项目！哪怕是"todo 清单"这种小项目，能让你遇到真实问题。
❌ 过度追求"完美"，一行代码反复修改
✅ 先让程序跑起来，再优化。完成比完美更重要，完美主义是学习进度的最大杀手。
🏆 给0基础小白的终极建议
  • 📌 每天至少敲代码30分钟，连续性比单次大量学习更重要，大脑需要重复才能形成记忆
  • 📌 学到哪个知识点就做对应的小项目，学完循环就做九九乘法表，学完文件就做日记本
  • 📌 遇到不会的先Google，再问AI，再看文档，自己解决的问题记得最牢
  • 📌 不要孤军奋战，加入学习群/开发者社区，有人一起学效率是单独学的3倍
  • 📌 3个月是关键坎，大多数人在这里放弃，坚持过去就会越来越顺，质变就在那一刻
  • 📌 用Python解决你真实遇到的问题，比如自动整理文件、处理表格，兴趣是最好的老师
