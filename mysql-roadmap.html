示例：完整的 CRUD 操作

示例：多表查询实战
-- 订单表 + 用户表 + 商品表 SELECT u.username, o.order_no, p.product_name, oi.quantity * p.price AS subtotal FROM orders o INNER JOIN users u ON o.user_id = u.id INNER JOIN order_items oi ON o.id = oi.order_id INNER JOIN products p ON oi.product_id = p.id WHERE u.status = 'active' ORDER BY o.created_at DESC; -- 子查询：查询消费超过1000的用户 SELECT username, email FROM users WHERE id IN ( SELECT DISTINCT user_id FROM orders GROUP BY user_id HAVING SUM(total_amount) > 1000 );

新手常见误区 & 避坑指南
❌ 光看不练，只看视频刷完就觉得会了
✅ 每学一个SQL语句立刻动手敲，自己跑一遍看结果。SQL是手艺，必须练。
❌ 不理解表关系，一顿乱JOIN
✅ 先画ER图理解表与表之间的关系，再写JOIN。乱JOIN是性能杀手。
❌ 忽视索引，认为SQL能跑就行
✅ 数据量大了没有索引会慢死。学索引要理解底层数据结构（B+树）。
❌ 不注意数据类型选择，用VARCHAR存数字
✅ 选错数据类型会影响性能和处理逻辑。数字用数字类型，日期用日期类型。
❌ 不理解事务，一顿乱更新
✅ 涉及多条数据修改时要考虑事务，否则数据会不一致。
❌ 只学SQL，不学数据库设计
✅ 会写SQL不算会数据库。表结构设计、索引优化、范式选择才是核心。
